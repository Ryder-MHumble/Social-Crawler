# B站自动发送私信脚本使用说明（并发版本）

## 功能说明

这个脚本可以自动读取 `openclaw_creators.csv` 文件中的博主信息，并使用 Playwright 浏览器自动化工具**并发**给多位博主发送私信，大幅提升效率。

### 🚀 并发特性

- **多标签页并发**：同时打开 5 个标签页发送私信
- **批次处理**：自动分批处理所有博主
- **效率提升**：相比串行发送，速度提升约 5 倍
- **智能延迟**：批次间自动延迟，避免触发反爬虫机制

## 安装依赖

```bash
# 安装 Playwright
pip install playwright

# 安装浏览器驱动
playwright install chromium
```

## 使用步骤

### 1. 运行脚本

```bash
python send_bilibili_dm.py
```

### 2. 手动登录

脚本会自动打开 Chrome 浏览器并访问 B站首页，你需要：
- 在浏览器中手动完成登录（扫码或密码登录）
- 登录成功后，脚本会自动检测并继续执行

### 3. 自动发送（并发模式）

登录成功后，脚本会：

- 将所有博主分成多个批次（每批 5 位）
- 每批次同时打开 5 个标签页并发发送
- 每个标签页独立访问博主主页、点击发消息、输入内容、发送
- 批次完成后等待 10 秒，继续下一批次
- 实时显示每个标签页的发送状态

### 4. 查看结果

脚本执行完成后会显示详细统计信息：

- 成功发送的数量
- 失败的数量
- 总计处理的博主数量
- 成功率百分比

## 并发配置

可以在脚本中修改并发数量：

```python
CONCURRENT_TABS = 5  # 同时打开的标签页数量（建议3-5个）
```

**建议值**：

- 3 个标签页：保守模式，适合网络较慢或担心被限制
- 5 个标签页：推荐模式，效率与安全的平衡
- 8 个标签页：激进模式，效率最高但风险较大

## 注意事项

### ⚠️ 重要提醒

1. **并发控制**：默认 5 个标签页并发，已经过测试，不易触发限制
2. **批次延迟**：每批次间自动延迟 10 秒，避免被检测
3. **账号安全**：建议分时段发送，不要一次性发送过多私信
4. **私信限制**：
   - 部分博主可能关闭了私信功能
   - B站可能对短时间内大量发送私信的行为进行限制
   - 建议每天发送不超过 100-150 条

5. **失败处理**：
   - 如果某位博主发送失败，脚本会记录并继续下一位
   - 失败原因可能包括：关闭私信、网络问题、页面结构变化等

6. **手动干预**：
   - 如果遇到验证码或其他安全检查，需要手动处理
   - 脚本会保持浏览器打开状态，方便你随时查看

## 分批发送建议

虽然脚本支持并发，但如果博主数量较多（如 200+ 位），仍建议分时段发送：

### 方法1：修改 CSV 文件

将 `openclaw_creators.csv` 分成多个文件：
- `openclaw_creators_batch1.csv` (前50位)
- `openclaw_creators_batch2.csv` (51-100位)
- ...

然后修改脚本中的 `csv_file` 变量，分批运行。

### 方法2：修改脚本添加范围参数

在脚本中添加起始和结束索引：

## 故障排查

### 问题1：找不到"发消息"按钮

**原因**：博主关闭了私信功能或页面结构变化

**解决**：脚本会自动跳过并记录失败

### 问题2：登录超时

**原因**：120秒内未完成登录

**解决**：重新运行脚本，更快完成登录操作

### 问题3：发送失败率高

**原因**：可能触发了 B站的反爬虫机制

**解决**：
- 增加延迟时间
- 分批发送
- 更换账号

### 问题4：浏览器无法启动

**原因**：Playwright 浏览器驱动未安装

**解决**：
```bash
playwright install chromium
```
