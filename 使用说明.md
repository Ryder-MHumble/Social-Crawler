# MediaCrawler 使用说明

## 🚀 快速开始

你已经配置好了以下平台的 Cookie：
- ✅ 小红书 (XHS)
- ✅ 抖音 (Douyin)
- ✅ Bilibili

### 直接运行

```bash
cd /Users/sunminghao/Desktop/MediaCrawler
python run.py
```

**完全自动化，无需任何确认！**

脚本会：
1. 自动检查 Cookie 配置
2. 逐个平台开始爬取
3. 如果某个平台的 Cookie 失效，会提示并自动跳过
4. 继续爬取其他平台

---

## 📋 爬取流程

```
检查 Cookie 配置
  ↓
爬取小红书 → 成功/失败（自动提示）
  ↓
爬取抖音 → 成功/失败（自动提示）
  ↓
爬取 B 站 → 成功/失败（自动提示）
  ↓
爬取微博 → 成功/失败（自动提示）
  ↓
完成！显示总结报告
```

**不会中断**：即使某个平台失败，也会继续爬取其他平台

---

## 💡 Cookie 失效提示

如果看到类似提示：
```
❌ [XHS] 小红书 爬取失败！
💡 提示: 小红书 的 Cookie 可能已失效，请更新 cookies_config.py
```

**解决方法**：
1. 打开 [cookies_config.py](cookies_config.py)
2. 按照 [COOKIE_GUIDE.md](COOKIE_GUIDE.md) 重新获取该平台的 Cookie
3. 更新对应平台的 Cookie
4. 重新运行 `python run.py`

## ⚙️ 配置修改

### 修改关键词

编辑 [run.py](run.py) 第 29 行：

```python
KEYWORDS = "中关村人工智能研究院,北京中关村学院"
```

改为你想要的关键词，用英文逗号分隔。

### 修改爬取平台

编辑 [run.py](run.py) 第 32 行：

```python
PLATFORMS = ["xhs", "dy", "bili", "wb"]
```

可选平台代码：
- `xhs` - 小红书
- `dy` - 抖音
- `bili` - Bilibili
- `wb` - 微博
- `ks` - 快手

## 🔧 常见问题

### Q: 运行时需要确认吗？
A: 不需要！直接运行 `python run.py`，全自动执行，不会有任何 y/n 确认提示。

### Q: 某个平台失败了会影响其他平台吗？
A: 不会！失败的平台会自动跳过，继续爬取其他平台。

### Q: Cookie 多久会过期？
A: 通常 15-30 天。过期后重新获取并更新 [cookies_config.py](cookies_config.py) 即可。

### Q: 如何知道 Cookie 是否有效？
A: 运行 `python run.py`，如果爬取成功就有效。失败会提示 "Cookie 可能已失效"。

### Q: 可以只爬取部分平台吗？
A: 可以！编辑 [run.py](run.py) 的 `PLATFORMS` 列表，只保留需要的平台。

## ⚠️ 重要提醒

1. **Cookie 安全**
   - Cookie 等同于登录密码，不要分享给他人
   - 不要上传到 GitHub 等公开平台
   - 已添加到 `.gitignore`，git 不会提交

2. **合规使用**
   - 仅供学习研究使用
   - 遵守平台使用规则
   - 控制爬取频率（建议间隔 1-2 小时以上）

3. **数据备份**

   - 建议配合定时任务自动备份
